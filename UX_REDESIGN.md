# SIGMA UX 평가 & 재설계 — "조용하고 설명 가능한" 방향

> 디자이너 + PM 합동 검토. 대상: 현재 UI(1~4.png). 기준: 우리가 만든 adaptive-engine 출력.
> 구현 아님 — 평가·목업·매핑 문서(팀 공유용).

## 1. 문제의식
현재 UI는 말풍선 챗봇은 아니지만 **"AI 코치/비서가 끊임없이 내레이션하는 대시보드"**다.
("Good morning, Director", "the algorithm suggests…", "You are 85% more productive",
명언 카드, 전면 모달 넛지, "SYSTEM STABLE".) 사용자가 "챗봇 같다"고 느낀 건 정확 —
**AI 페르소나가 화면을 지배**한다. 지능은 좋은데 *공연(performance)*이 과하다.

## 2. 편의성 평가
🟢 살릴 것: 최우선 1개 강조(결정마비↓) · 좀비 넛지·에너지 인지(차별점) · 카드/다크모드.
🔴 편의성을 해치는 것:
1. **AI 페르소나 과잉** — 매 화면 코치가 말 검. 신기함은 3일이면 닳고 마찰만 남음. "Director"·명언은 프로 사용자엔 유치/거슬림.
2. **전면 모달 넛지(2.png)** — 저관여 결정("보관?")에 화면 전체를 멈춤 = 흐름 끊김.
3. **자신만만한 블랙박스** — "85% 더 생산적", "Expected Priority: High"인데 *왜*도 *교정 수단*도 없음 → 불신.
4. **밀도 과다(4.png)** — 인사이트+챌린지+브리핑+타이머+명언이 경쟁, 정작 "지금 뭐?"가 묻힘.
5. **상태 입력이 게이트(1.png)** — 매번 에너지+가용시간을 세팅해야 우선순위가 나옴 → 대부분 안 함.
6. **근거 없는 숫자(68%/85%)** — 실제와 어긋나면 신뢰 붕괴.

### 핵심 진단 (한 줄)
일상 to-do의 핵심 루프(목록 보고 → 하나 고르고 → 하고 → 완료)는 **빠르고 조용해야** 하는데,
AI가 계속 말 걸어 **느리고 시끄럽다.** → **AI는 결정에 가치를 더할 때만, 맥락에서, 한 번에 하나씩.**

## 3. 개선 8원칙
1. **페르소나 절제** — 상시 내레이션/명언 제거. AI 보이스는 결정 지점(시작 추천 1줄·좀비 넛지)에만.
2. **전면 모달 → 인라인 넛지** — 행 칩(보관/쪼개기/계속) + 스와이프·실행취소.
3. **한눈에 + 탭하면 설명** — 리스트에 긴급도 점 + 한 줄 사유, 탭하면 점수 요소분해.
4. **상태 입력은 게이트가 아니라 옵션** — 시간대/캘린더 추론을 기본값, 빠른 조정만.
5. **주장엔 근거** — "85%"는 탭하면 데이터로, 아니면 제거.
6. **화면당 한 가지 일** — Today=지금 뭐(리스트+추천1). 타이머·명언·주간통계는 Analytics로.
7. **톤을 청중에 맞춤** — 프로 타깃이면 차분·중립, 게이미피케이션은 옵트인.
8. **통제 = 신뢰** — 핀(고정)·"왜 이 순위?" 펼치기·AI 교정 수단.

## 5. 엔진 출력 ↔ UI 매핑
| 엔진 출력(TaskResponse) | 개선에서의 쓰임 | UI 요소 |
|---|---|---|
| `priorityScore` | 랭킹·"지금 이거" 1순위 | 상단 추천 + 정렬 |
| `urgencyLevel`(RED/YELLOW/GREEN/NONE/STALE) | 행별 신호등 점·마감없음/방치 구분 | 🔴🟡🟢⚪ 점 |
| `reason` | "왜 1위?" 한 줄·넛지 문구 | 추천 줄 보조 텍스트 |
| `importanceScore/urgencyScore/delayPenalty` | "왜 1위? ▾" 펼침 | 점수분해 팝오버 |
| `isExploration` (+사유) | "발견/오늘의 챌린지" 1개 | 하단 탐색 배너 |
| `isZombie` / `stuckLevel`(AGING/STUCK/STALLED) | "묵은 일" 인라인 넛지 등급 | 행 칩 |
| `isPinned` | 핀 고정(override) | 📌 아이콘 |
| `WeightChangeLog` / `resetWeights` (거버넌스) | "패턴 학습/되돌리기" | History 하단 |

→ 개선 대부분이 **이미 만든 엔진 출력**으로 구현 가능(적은 UI 작업, 큰 신뢰·통제 회복).

## 6. 미해결 · 조율 필요
- UI의 **에너지(LOW/MED/HIGH) + 가용시간 입력**은 현 adaptive-engine이 안 받음 →
  로드맵 **항목2(effort)·8(energy)** 가 있어야 "Expected Priority"·"morning energy peak" 흐름이 진짜로 작동.
  입력 계약 변경이라 **본체(data-pipeline) 팀과 조율** 필요.

## 한 줄
**같은 비전이지만 — 엔진은 UI가 기대하는 입력(에너지/용량)을 아직 안 받고, UI는 엔진이 주는 출력(설명/등급/통제)을 아직 안 보여준다. "조용하지만 설명 가능한" UI가 이 엔진과 가장 잘 맞는다.**
